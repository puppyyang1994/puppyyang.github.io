Git的基本使用

  1. git bash 然后输入 mkdir coding 就创建了一个文件夹 （此时文件夹里是空的）

  2. cd coding 到文件夹coding的位置

  3. 再git init 此时文件夹中会出来一个.git的文件夹

  4. 在coding文件夹中手动创建一个test.txt

  5. 使用命令行 git add test.txt 将文件放到暂存区

  6. 再git status查看文件夹的状态 会显示 changes to be committed (绿色的 new file test.txt

  7. 然后git commit -m “第一次提交” 会显示 creat mode test.txt 这样文件就放到本地仓库了

  8. git remote add origin(名) url

  9. git branch -M main

  10. git push -u origin main

    如果再上传一个文件夹的话 要把origin改成别的名字

  11. 把文件内容改了之后 再git status 会发现 变成了modified 此时 我们git add . 添加到暂存区 然后 git commit -m “second commit” 然后 git push -u origin main

注意：- 如果一个文件夹 已经有.git文件包了 不能在新建一个有.git的文件包进来

​ - git 不能远程上传一个空文件夹 必须在它的里面放点东西

返回上一级文件夹 命令行 cd ..

分支主要是每个程序元负责自己的版块 等到最后大家都合并到master分支上

  1. 通过git branch 命令行 可以查看所有的分支，分支前的*号表示当前所处的分支

  2. git branch login 基于当前的main分支又创建了一个login分支 分支中的代码和main一致

  3. 通过chekout 来切换到指定分支上 git checkout login

  4. 通过git checkout -b logout 可以新建logout分支 并切换到它的上面

  5. 合并分支：先切换到main 分支上 在main分支上运行 git merge login命令 将login代码合并到main上

  6. 合并之后就删除本地的login 分支 git branch -d login

  7. 第一次上传分支 git push origin logout (先要切换到这个分支上来) origin是指远程仓库的别名 ：第一次推送分支需要带 -u 参数，此后可以直接使用 git push 推送代码到远程分支。

  8. 通过git remote show 可以查看所有分支列表、

  9. 跟踪分支：把远程仓库的分支下载到本地仓库 git checkout logout（给本地分支重命名）

    git checkout -b newbranch origin/logout(远程分支名称)

  10. 从远程分支拉取代码 git pull 保持远程的代码和本地仓库的一致

  11. 删除远程分支 git push origin –delete 远程分支名
