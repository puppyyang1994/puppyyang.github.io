  • Object.defineProperty方法会直接在一个对象上定义一个新属性，或者修改一个对象的现有属性，并返回此对象. 应当直接在 Object 构造器对象上调用此方法，而不是在任意一个 Object 类型的实例上调用。

  • 数据代理：通过一个对象代理对另一个对象中的属性进行读、写操作

  • 数据代理。

    1. Vue中的数据代理： 通过vm对象来代理data对象中属性的操作（读/写）

    2. Vue中数据代理的好处： 更加方便的操作data中的数据

    3. 基本原理： 通过Object.defineProperty()把data对象中所有属性添加到vm上。 为每一个添加到vm上的属性，都指定一个getter/setter。 在getter/setter内部去操作（读/写）data中对应的属性。

总结：

  1. computed是Vue的计算属性，是根据依赖关系进行缓存的计算，只有在他的相关依赖关系发生改变时才会进行更新，通过getter和setter实现对属性数据的显示和监视。计算属性存在缓存，多次读取只执行一次getter. 计算属性本质上还是属性，写法如函数，命名如属性。在处理一些复杂的逻辑时计算属性很有用。
  2. computed计算属性的值我们需要用到它，依赖变化运算过程自动执行；watch监听的值我们不需要用到它，我们只是想在它监听的值改变了的时候搞点事情–注册用户的时候是否可用

    总结数据监测：

    1. Vue会监视data中所有层次的属性
    2. 如何监测对象中的数据：
      • 通过setter实现监视，且要在new Vue时就要传入要检要监测的数据。
        • 对象中后追加的属性，Vue默认不做响应处理
        • 如需给后添加的属性做响应式，请使用API：Vue.set(target, propertyName/index, value)或Vue.$set(target, propertyName/index, value)
    3. 如何监测数组中的数据：
      • 通过包裹数组更新元素的方法实现，本质就是做了两件事：
        • 调用原生的对应的方法进行数组更新
        • 重新解析模板，进行页面更新
    4. 在Vue修改数组中的某个元素时一定要用如下方法：1. 使用这些API：push, pop, shift, unshift, splice, sort, reverse; 2. Vue.set()或vm.$set()
    5. 特别注意：Vue.set() 和 vm.$set() 不能给vm 或 vm的根数据对象 添加属性！！！
