Vue学习笔记4

1. vue/cli 如何创建脚手架项目和启动

  1. @vue/cli是Vue官方提供的一个全局模块包，此包用于创建脚手架项目

  2. 脚手架是一个工作平台

  3. 脚手架开箱即用 0配置； babel支持， css/less支持, 开发服务器支持

  4. 用node安装一个全局包 npm install @vue/cli -g 全局包会在计算机中配置全局命令（vue命令）

  5. 如何创建一套脚手架项目

    • 创建项目vue create vuecli-demo
    • 选择模板和包管理器(Vue 2) 等待创建完毕
    • 按照提示 运行 cd 文件名 + npm run serve 就成功了

  6. 各个文件包的含义

    • node_modules 项目依赖的第三方包
    • public 静态文件目录
      • favicon.ico 浏览器小图标
      • index.html 单页面的html(网页浏览的是它)
    • src 业务文件夹
      • assets静态资源
        • logo.png vue的logo图片
      • components 组件目录
        • hello.vue 欢迎页面vue代码文件
      • App.vue 整个应用的根组件
      • main.js 入口的js文件（webpack打包的入口)
    • .gitignore git提交忽略配置
    • babel.config.js babel配置
    • package.json 依赖包列表
    • README.md 项目说明
    • node.lock 项目包版本锁定和缓存地址

  7. 项目入口和各文件的关系

    • 入口 main.js
    • new Vue(Vue 实例化) 同时插入html <div id="app"></div>(index.html)
    • 根组件 App.vue （标签样式数据）Vue页面主入口
    • 创建组件1,2,3.。。。

    App.vue=>main.js=>index.html

  8. vue.config.js 配置文件

    eslint是代码检查工具, 违反规定就报错

    transpileDependencies: true, lintOnSave:false, 重启服务器，即可关闭语法检查

2. 单文件组件

  1. Vue推荐用.vue文件来开发项目
  2. template里只能有一个根标签
  3. vue文件-独立模块-作用域互不影响
  4. style配合scoped属性，保证样式只针对当前的template内标签生效
  5. vue文件配合webpack，把它们打包插入到index.html中

4.路由

4.1 路由

  1. 理解： 一个路由（route)就是一组映射关系（key-value), 多个路由需要router路由器进行管理

  2. 前端路由：key是路径， value是组件

  3. 了解：

  4. vue-router vue的一个插件库，专门用来实现SPA

  5. SPA应用：

    • 单页面Web应用
    • 整个应用只有一个完整的页面
    • 点击页面中的导航链接不会刷新页面，只会做页面的局部更新
    • 数据需要通过ajax请求获取

  6. 基本使用

    • 安装vue-router npm i vue-router@3

    • 引入import VueRouter from 'vue-router' (main.js文件) 并使用

    • 编写路由配置项（router文件包 index.js)
