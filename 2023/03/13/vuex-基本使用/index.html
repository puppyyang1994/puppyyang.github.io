vuex 基本使用

1.Vuex的由来

  • Vuex是一个专为Vue.js应用程序开发的状态管理模式。采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

  • 由于单页面应用，组件化开发。组件间的通信：
    父给子props ; 子给父$emit(自定义事件) ; 兄弟间的组件 (Eventbus 公共的$emit); 非关系型组件间的数据共享通过Vuex (是一个状态共享的框架）

  • Vuex是采用集中式管理组件依赖的共享数据的一个工具。

2. 基本用法

  1. State 存放数据
  2. Mutations改数据 只能写同步代码 (这里不能进行异步的操作 ajax 定时器等异步操作)
  3. Actions 可以异步操作
  4. getters (相当于vuex的计算属性 处理state)

简而言之：State存储共享状态数据，修改state必须通过mutations完成共享状态数据的修改，只能执行同步代码; actions可以进行异步操作， 所得到的的异步操作的数据也必须通过【mutations】改到【state】里
和组件的关系：state中的数据可以在组件中显示，组件可以调用【action】

3. Vuex初始化的步骤

  1. 建立一个脚手架项目 vue create vuexdemo
  2. 初始化： npm i vuex –save 安装到运行时的依赖 （就是项目上线后依然需要的依赖） 开发时依赖 是指调试时的依赖 cd vuexdemo 然后 npm i vuex@3 –save (注意版本的问题 vue2 安装vuex@3 vue3安装vuex@4)
  3. main.js中 引入 Vuex import Vuex from ‘vuex’ Vue.use(Vuex)注册 实例化Vuex New Vue中配置 store
    const store = new Vuex.Store()
  4. 记得去vue配置文件配置：lintOnSave:false
  5. state是存放公共状态数据的地方
    定义一个state(对象）

如何应用？

  1. 定义Mutations （main.js中）

  • 数据快照：一次mutation的执行，立刻得到一种视图状态，因为是立刻，所以必须是同步（同步代码永远都是先执行的）

  • 定义mutations (是一个对象) 每一个mutation方法都有对应的参数 第一个参数:state 指的就是当前vuex中的state对象； 第二个参数 payload 载荷 提交mutation的方法的时候 传递的参数，他可以是任何形式的任何类型的值（不需要传可以不传）

  • 去组件中调用mutations:

  1. 异步代码放到actions中
  1. getters
  1. Vuex中的模块化 module

由于使用单一状态树，应用的所有状态会集中到一个比较大的对象。当应用变得非常复杂时，store 对象就有可能变得相当臃肿。
由此产生了模块化

  • 模块化里的命名空间namespaced

默认情况下 模块内部的action mutation 和 getter是注册在全局命名空间的， 这样可以使得多个模块能够对同一mutation或者action作出响应（都可以通过$store调用到）,多个模块间没有阻碍的调用
